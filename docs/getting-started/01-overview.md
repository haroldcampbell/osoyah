# 01 Overview

Purpose
- Spec-driven development for a kanban-style board app.
- Changes are incremental, auditable, and recoverable.

Where to start
- Read `AGENTS.md`, `docs/process.md`, `docs/principles.md`, `docs/decisions.md`.
- Read `session-hand-off.md` and the latest file in `session-hand-offs/`.

Quickstart flow
- Pick the next spec in `specs/`.
- Review scope, DoD, and acceptance tests.
- Implement changes and update docs.
- Run acceptance tests.
- Update DoD, milestone checklist, and hand-off.
